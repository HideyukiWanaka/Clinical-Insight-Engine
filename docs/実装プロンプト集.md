# CIE 実装プロンプト集

> **使い方**: 各プロンプトをAIコーディングアシスタント（Claude / Cursor / Copilot等）に貼り付けて実装を進める。  
> 各プロンプトは `SRD_完全版.md` を参照するため、AIが読める場所に置くこと。  
> 実装は Phase 0 → Phase 6 の順で進める。前フェーズの完了を確認してから次に進むこと。

```

### P1-1: ファイルアップロードAPI（日本語・文字コード対応）

```
あなたはExpert Backend Developerです。
SRD_完全版.md の「4. Feature 1: データ入力・前処理パイプライン」を参照し、
CSV/Excel のアップロードAPIを実装してください。

## タスク
1. POST /api/v1/data/upload エンドポイントを実装してください:
   - multipart/form-data でCSV/Excel（.csv, .xlsx, .xls）を受け付ける
   - ファイルサイズ上限: 100MB
   - 対応エンコーディング: UTF-8, Shift-JIS, CP932（chardet で自動判定）
   
2. アップロード後の処理:
   - pandas でデータを読み込む（日本語列名を正確に保持）
   - 列型を自動推論する（numeric / factor / date / character）
   - ColumnMeta 配列を生成する（SRD の型定義参照）
   - 生データをAES暗号化してRedisに保存（TTL: 86400秒）
   - session_id を発行してクライアントに返す

3. レスポンス形式:
   ```json
   {
     "session_id": "uuid",
     "row_count": 500,
     "column_metadata": [
       {"name": "年齢", "type": "numeric", "missingCount": 5, "missingRate": 0.01},
       ...
     ],
     "sheets": ["Sheet1", "Sheet2"]  // Excelの場合のみ
   }
   ```

## セキュリティ制約（SRD Rule 1）
- 生データはRedis以外に保存しない
- ColumnMetadata（列名・型・欠損率）のみDBに保存可
- Redisキーはsession_idとする

## 完了条件
- 日本語列名のCSVが正しく読み込める
- UTF-8とShift-JISが自動判定される
- Excelファイルのシート一覧が返される
- 100MB超は適切なエラーを返す
```

### P2-2: Analysis Skills 実装（全11種）

```
あなたはExpert R Developerです。
SRD_完全版.md の「3. Analysis Skills 一覧」を参照し、
以下の全11スキルを r-engine/skills/ に実装してください。

## 共通ルール（全スキルに適用）
- ファイル名: {skill_id}.R
- 関数は execute_{skill_id}(df, parameters) の形式
- 実行したRコードをコード文字列として変数に蓄積し、r_code として返す
- 日本語列名はバッククォートで囲む（例: df$`年齢`）
- グラフはPNG（デフォルト）またはTIFF（DPI指定可、300または600）でbase64エンコードして返す

## 実装するスキル（各スキルのパラメータ仕様も定義すること）

### 1. table1_generator
- parameters: vars（列名リスト）, strata（群分け変数, optional）, test（TRUE/FALSE）
- output: HTML文字列としてのTable 1

### 2. chi_square_fisher
- parameters: var1, var2（カテゴリ変数の列名）
- 自動判定: 期待度数<5のセルが存在するときFisher's exact testを使用

### 3. ttest_mannwhitney
- parameters: continuous_var, group_var
- 自動判定: normality_checkの結果を受けてt検定またはMann-Whitney Uを選択

### 4. normality_check
- parameters: vars（連続変数列名リスト）
- output: Shapiro-Wilk検定結果 + ヒストグラム + Q-Qプロット（変数ごとに1枚）

### 5. logistic_regression
- parameters: outcome（2値変数）, predictors（説明変数リスト）, adjust_vars（調整変数, optional）
- output: OR, 95%CI, p値のテーブル + forest plot

### 6. linear_regression
- parameters: outcome（連続変数）, predictors, adjust_vars（optional）
- output: β係数, 95%CI, p値, R²のテーブル

### 7. kaplan_meier
- parameters: time_var, event_var, group_var（optional）
- output: KM曲線（ggplot2）+ Log-rank検定結果

### 8. cox_regression
- parameters: time_var, event_var, predictors, adjust_vars（optional）
- output: HR, 95%CI, p値のテーブル + forest plot

### 9. correlation_analysis
- parameters: vars（連続変数リスト）, method（"pearson"|"spearman"）
- output: 相関係数行列テーブル + ヒートマップ

### 10. roc_auc
- parameters: outcome, predictor
- output: AUC（95%CI）+ ROC曲線（ggplot2）

### 11. forest_plot
- parameters: estimates（OR/HR/β値リスト）, ci_lower, ci_upper, labels, reference_line
- output: フォレストプロット（ggplot2）

### 12. sample_size_calc
- parameters: test_type（"t_test"|"chi_square"|"correlation"|"logistic"|"survival"）,
              alpha, power, effect_size（またはその他検定固有パラメータ）
- output: 必要サンプルサイズ + 計算式の説明テキスト

## 完了条件
- 全スキルがサンプルデータで正常実行できる
- r_code フィールドに実行したRコード全文が含まれる
- RStudioで r_code を実行すると同じ結果が得られる
- グラフがbase64形式で返される
```
