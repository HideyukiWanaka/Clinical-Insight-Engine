# Stat Consultant — インストールと使い方（macOS 版）

統計解析の相談ができるチャットです。相談すると、根拠となる考え方と一緒に
**R のコード**が返ってきて、ボタン1つで RStudio のカーソル位置に挿入できます。
RStudio に読み込んでいるデータの構造（列名・型・欠損数など）は自動で共有される
ので、毎回説明し直す必要はありません。

> **お使いのデータの中身（行データ）が送信されることはありません。**
> 詳しくは「データの取り扱い」を参照してください。

---

## 1. 必要なもの

| | |
|---|---|
| macOS | Apple Silicon（M1 以降） |
| R | 4.1 以降 |
| RStudio | 最新版を推奨 |
| API キー | 3社のいずれか1つ（下記 §3） |

Python や Node.js は**不要**です。同梱されています。

---

## 2. インストール

### 手順1 — zip を展開する

ダウンロードした `stat-consultant-macos-arm64.zip` をダブルクリックして展開します。

展開先はどこでも構いませんが、**あとで動かさない場所**にしてください
（インストール時にその場所を記録するため）。`アプリケーション` フォルダや
`書類` フォルダの中がおすすめです。

### 手順2 — RStudio でインストーラを実行する

RStudio を開き、**コンソール**（左下のペイン）に次の1行を貼り付けて Enter を押します。
`<展開したフォルダ>` の部分は、手順1で展開したフォルダのパスに置き換えてください。

```r
source("<展開したフォルダ>/install.R")
```

> **パスの入力が面倒なときは:** `source("` まで入力したあと、Finder から
> フォルダ内の `install.R` をコンソールにドラッグ＆ドロップすると、
> パスが自動で入力されます。最後に `")` を足して Enter を押してください。

必要なパッケージの導入が始まります（初回は数分かかることがあります）。
最後に次の表示が出れば成功です。

```
[Stat Consultant] インストール完了。
```

### 手順3 — RStudio を再起動する

再起動すると、メニューバーの **Addins** に「Stat Consultant」の項目が現れます。

---

## 3. API キーを設定する

このアプリは**利用者ご自身の API キー**を使います（キーは同梱されていません）。
料金は各社からご自身に直接請求されます。

### キーの取得先

| 提供元 | 取得先 |
|---|---|
| Anthropic（Claude） | https://console.anthropic.com |
| OpenAI（GPT） | https://platform.openai.com |
| Google（Gemini） | https://aistudio.google.com |

どれか1つで動きます。迷う場合は Anthropic（Claude）または Google（Gemini）を
お使いください。

### 設定手順

1. Addins →「**Stat Consultant: 起動**」を選ぶ（ブラウザでチャット画面が開きます）
2. 画面右上の**歯車アイコン**をクリック
3. 使う提供元の欄にキーを貼り付けて保存

キーは **macOS のキーチェーン**に保存されます。アプリの設定ファイルや
テキストファイルには保存されません。

## 4. 基本の使い方

### 起動と終了

| したいこと | 操作 |
|---|---|
| 起動する | Addins →「Stat Consultant: 起動」 |
| 終了する | Addins →「Stat Consultant: 終了」 |

「起動」を押すと、裏側のプログラムが立ち上がり、ブラウザにチャット画面が開き、
RStudio との連携が始まります。**普段はこの2つだけ使えば十分です。**

（「ポーリングのみ開始／停止」は、チャット画面は開いたまま RStudio への
コード挿入だけを一時的に止めたいときに使います。通常は不要です。）

### 相談する

画面下の「**統計の相談を入力…**」に質問を書いて「**送信**」を押します。

```
対応のある2群の比較をしたい。前後で血圧を測っています。
```

```
この解析で多重比較の補正は必要ですか
```

```
生存時間解析をしたいのですが、どの手法が適切ですか
```

返答は次の形で表示されます。

- **一言の理由** — なぜその手法なのか
- **詳細を見る ▼** — 前提条件や注意点（クリックで展開）
- **R コード** — そのまま実行できる形

### コードを RStudio に送る

コードの右上にある「**RStudio へ送る**」ボタンを押すと、RStudio で開いている
スクリプトの**カーソル位置**にコードが挿入されます。

> **反応がないときは:** RStudio でスクリプト（.R ファイル）を開いているか
> 確認してください。コンソールしか開いていないと、コードが失われないよう
> 挿入を保留します。スクリプトを開けば挿入されます。

隣の**コピーボタン**でクリップボードにコピーすることもできます。

### モデルを選ぶ

画面上部の「**モデルを選択**」で使用するモデルを変更できます。
既定では、設定済みの提供元の中から高性能なモデルが自動で選ばれます。

### 新しい相談を始める

「**新しいチャット**」を押すと会話がリセットされます。
話題が大きく変わるときに使うと、文脈の混線を防げます。

## 5. データについて説明しなくてよい理由

RStudio に読み込んでいるデータの**構造**は自動的に共有されています。

たとえば `df` というデータフレームがあれば、次が伝わります。

- データフレーム名と行数
- 各列の名前・型・欠損値の個数
- カテゴリ列については、**水準の個数と各グループの件数だけ**（例:「2群、45件と43件」）

そのため「どんな列があるか」「群のサイズは揃っているか」を毎回説明する必要はなく、
**実際の列名を使って**相談できます。
データを読み込み直したり変数を作ったりすると、自動的に反映されます。

> **水準の「名前」は送信されません。** たとえば `群` という列が `治療群` と
> `対照群` に分かれていても、伝わるのは「2群、45件と43件」までで、
> `治療群` という文字列そのものは送られません（§7）。どちらを基準にしたいかなど
> 水準名が関わる相談では、文章のほうに書き添えてください。

## 6. 参考資料・図を渡す

入力欄の**クリップアイコン**（ファイルを添付）から2種類を渡せます。

| 種類 | 形式 | 扱い |
|---|---|---|
| **参考資料** | Markdown / テキスト / PDF | 保存され、以降の相談で参照されます |
| **参考図** | 画像（PNG / JPEG / GIF / WebP） | その回の相談限りで、保存されません |

**参考資料**は、院内の解析方針やガイドラインを渡しておくのに向いています。
たとえば「当院では有意水準を 0.01 に設定する」と書いた文書を登録しておくと、
以降の回答がその方針を踏まえたものになります。

**参考図**は、論文の図やグラフについてその場で質問したいときに使います。

> PDF は**テキスト情報を含むもの**に限ります。スキャンした画像だけの PDF は
> 文字を読み取れません。

## 7. データの取り扱い

臨床データを扱うため、次の設計になっています。

### 送信されないもの

- **行データ（実際の測定値・患者データ）は一切送信されません。**
- **カテゴリの水準名そのもの**（`治療群`、`A病院` など、セルに入っている
  文字列）**も送信されません。** 送られるのは水準の個数と各グループの件数という
  数値だけです。
- 個人情報を含む可能性のある列は、**列ごとまるごと除外**されます。
  氏名・患者ID・生年月日・電話番号・住所・メールアドレスなどの列名が対象です。
- 除外は **RStudio 側で送信前に**行われ、受信側でも再度検査されます（二重化）。

### 送信されるもの

- データフレーム名、行数
- 列名、型、欠損値の個数
- カテゴリ列の水準の個数と、各グループの件数（数値のみ）
- ご自身が入力した相談文、添付した参考資料・参考図

### 通信先

- 裏側のプログラムは**この Mac の中だけ**で動き、外部からは接続できません。
- 外部に出るのは、ご自身が設定した API キーの提供元（Anthropic / OpenAI /
  Google）への相談内容のみです。

> **ご注意:** 相談文に患者情報を直接書き込むと、それは送信されます。
> 自動除外の対象はデータフレームの構造情報であり、入力欄の文章ではありません。

## 8. 困ったとき

### 「開発元を検証できないため開けません」と出る

インストーラが自動で対処しますが、それでも出る場合は、
ターミナル（アプリケーション → ユーティリティ）で次を実行してください。

```
xattr -dr com.apple.quarantine "<展開したフォルダ>/stat-consultant-backend"
```

### キーチェーンの確認ダイアログが出る

「**stat-consultant-backend がキーチェーンに保存されている情報を使おうとしています**」
というダイアログが出たら、「**常に許可**」を押してください。

これは **アプリを更新した直後に1回だけ**出ます（macOS が更新後のアプリを
別のアプリとして扱うためです）。異常ではありません。
ダイアログを閉じずに放置すると、モデル一覧が空のままになります。

### モデル一覧が空になる

次の順に確認してください。

1. API キーが設定されているか（歯車アイコン →「✓ 設定済み」の表示）
2. 上記のキーチェーンのダイアログが出ていないか
3. インターネットに接続されているか（院内ネットワークの制限を含む）

### 「バックエンドが応答しません」と出る

ログに原因が記録されています。次を実行して内容をご確認ください。

```r
file.show("~/.stat-consultant/backend.log")
```

よくある原因は、ポート 8000 が別のアプリに使われていることです。

### 画面右上が「未接続」のまま

Addins →「Stat Consultant: 終了」→「Stat Consultant: 起動」の順に
実行し直してください。

### インストールした場所を動かしてしまった

`install.R` をもう一度実行すれば、新しい場所が記録されます。

---

## 9. 更新のしかた

1. 新しい zip を展開する（古いフォルダは削除して構いません）
2. RStudio で `source("<新しいフォルダ>/install.R")` を実行する
3. RStudio を再起動する

**API キーは再設定不要です**（キーチェーンに残ります）。
更新後の初回起動時にキーチェーンの確認ダイアログが1回出ますので、
「常に許可」を押してください。

---

## 10. 保存される場所

すべてご自身のホームフォルダ配下にあります。

```
~/.stat-consultant/
├── config.json        インストール場所の記録
├── rstudio_token      RStudio 連携用（起動ごとに再生成）
├── backend.log        動作ログ（不具合時の確認用）
├── conversations/     会話の履歴
└── references/        アップロードした参考資料
```

アンインストールする場合は、このフォルダと展開したフォルダを削除し、
RStudio のコンソールで `remove.packages("statConsultantAddin")` を実行してください。
