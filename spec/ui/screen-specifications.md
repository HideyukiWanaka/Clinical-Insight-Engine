## SCR-01: プロジェクトダッシュボード

### 概要

| 項目 | 内容 |
|------|------|
| URL/ルート | `/` |
| 役割 | プロジェクト一覧と新規作成エントリポイント |
| 初期表示状態 | 既存プロジェクト一覧（最終更新順） |

### レイアウト

```
┌──────────────────────────────────────────────────────────┐
│ [ステータスバー]                                          │
├──────────────────────────────────────────────────────────┤
│                                                          │
│  CIE Platform          [+ 新規プロジェクト]              │
│                                                          │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐     │
│  │ ProjectCard │  │ ProjectCard │  │ ProjectCard │     │
│  │             │  │             │  │             │     │
│  │ ✓ 完了      │  │ ⟳ 実行中   │  │ 🟣 承認待ち │     │
│  │ 品質: 94    │  │ ステップ3/7 │  │ 1件承認待ち │     │
│  └─────────────┘  └─────────────┘  └─────────────┘     │
│                                                          │
│  [最近のアクティビティ]                                   │
└──────────────────────────────────────────────────────────┘
```

### コンポーネント配置

| コンポーネント | 配置 | 説明 |
|-------------|------|------|
| `ProjectCard` | メイングリッド（3列） | 1プロジェクト = 1カード |
| `WorkflowStatusBadge` | 各カード右上 | pending/running/completed/failed/waiting_for_human |
| `ApprovalCounter` | 各カード | 承認待ちタスク数（0の場合は非表示） |
| `QualityScoreBadge` | 各カード下部 | 完了プロジェクトのみ表示 |
| `NewProjectButton` | ヘッダー右 | クリックでSCR-02へ遷移 |
| `ActivityFeed` | ページ下部 | 直近のシステムアクティビティ（5件） |

### ProjectCardの状態別スタイル

| ワークフロー状態 | カード左ボーダー色 | バッジ |
|---------------|----------------|--------|
| completed | `--cie-success` | ✓ 完了 |
| running | `--cie-blue-500` | ⟳ 実行中 |
| waiting_for_human | `--cie-approval` | 🟣 承認待ち |
| failed | `--cie-critical` | ✕ 失敗 |
| draft / planned | `--cie-gray-200` | ○ 準備中 |

### 承認待ちプロジェクトの優先表示

`waiting_for_human`状態のプロジェクトはグリッドの先頭に表示する。
ページ上部に「承認待ちプロジェクトが N 件あります」のバナーを表示する。

## SCR-02: 研究意図入力画面

### 概要

| 項目 | 内容 |
|------|------|
| URL/ルート | `/projects/new` |
| 役割 | 自然言語による研究意図の入力とPlanner Agent実行 |
| 人間承認ポイント | intent_object確認・承認（UP-002） |

### レイアウト

```
┌────────────────┬──────────────────────────────┬────────────────────┐
│ 左ペイン       │ 中央ペイン                   │ 右ペイン           │
│                │                              │                    │
│ ▸ 新規         │  研究目的を入力してください  │ 🤖 AI解釈結果     │
│   プロジェクト │  ┌──────────────────────┐   │                    │
│                │  │                      │   │ objective:         │
│                │  │  IntentTextarea      │   │  between_group_... │
│                │  │  (200px以上)         │   │                    │
│                │  │                      │   │ outcome_type:      │
│                │  └──────────────────────┘   │  continuous ✓      │
│                │                              │                    │
│                │  ┌──────────────────────┐   │ paired: ?          │
│                │  │  DatasetDropzone     │   │  ⚠ 確認が必要     │
│                │  │  CSVをドロップ       │   │                    │
│                │  └──────────────────────┘   │ [intent修正]       │
│                │                              │                    │
│                │  [解析を開始する →]          │ confidence: 0.82   │
└────────────────┴──────────────────────────────┴────────────────────┘
```

### 中央ペインの詳細

**IntentTextarea:**
- 最小高さ: 200px（自動拡張）
- プレースホルダー:
  ```
  研究目的を自然な言葉で記述してください。

  例）「治療群Aと対照群Bの術後90日死亡率を比較したい」
      「BMIと血圧の相関を調べたい」
      「介入前後の痛みスコアを同一患者で比較したい」
  ```
- 入力中（500msデバウンス後）にPlanner Agentへリクエストを送信
- フォントサイズ: 16px（読みやすさ優先）

**DatasetDropzone:**
- 受付形式: CSV, TSV, XLSX（ファイルサイズ上限: 100MB）
- ドラッグ中: 背景 `--cie-blue-100`、ボーダー `--cie-blue-500`（破線→実線）
- アップロード後: ファイル名・行数・列数をサマリー表示
- **PIIリスク警告:** アップロード後に「このデータは安全に処理されます。
  raw dataはAIコンテキストに注入されません。」のインフォバナーを表示

### 右ペインの詳細（AI解釈結果パネル）

- Planner Agentの処理中: スピナー + 「意図を解析中...」
- 解析完了後: intent_objectの各フィールドを日本語ラベル付きで表示
- confidence_score:
  - 0.8以上: 緑ドット + 値
  - 0.6〜0.79: 黄ドット + 値 + 「確認推奨」
  - 0.6未満: 赤ドット + 「人間による確認が必要です」
- `paired=null`や曖昧フィールドは黄色ハイライト + 「確認が必要」ラベル

### 承認フロー

「解析を開始する」ボタンクリック時:
1. intent_objectが確定していない場合: 「未確認の項目があります。確認してください」
2. 確定している場合: 承認パネル（UP-002規則に従う）を表示
3. 承認後: SCR-03 ワークフロービジュアライザーへ遷移

## SCR-03: ワークフロービジュアライザー

### 概要

| 項目 | 内容 |
|------|------|
| URL/ルート | `/projects/:id/workflow` |
| 役割 | ワークフロー全体の可視化と進捗モニタリング |
| 人間承認ポイント | 各承認ノードで右ペインから承認操作 |

### レイアウト

```
┌────────────────┬──────────────────────────────┬─────────────────────┐
│ 左ペイン       │ 中央ペイン                   │ 右ペイン            │
│                │                              │                     │
│ ▸ プロジェクト │  ワークフロー進捗            │ 🟣 HUMAN APPROVAL  │
│   概要         │                              │    REQUIRED         │
│                │  ①intake  ②validate  ③...  │                     │
│ ▸ ステップ一覧 │  ✓        ⟳          ○     │  Rスクリプトの実行  │
│   ① intake    │                              │  を承認します。     │
│   ② validate ←│  [現在: validate_dataset]    │                     │
│   ③ classify  │  data_quality agent 実行中   │  □ 確認しました    │
│   ...         │  (15秒経過)                  │                     │
│                │                              │  [キャンセル][承認] │
│ ▸ 成果物       │  [▼ ステップ詳細を表示]     │                     │
│                │                              │ AgentActivity:      │
└────────────────┴──────────────────────────────┴─────────────────────┘
```

### ワークフローDAGの表示仕様

**ステップの表示単位:**
spec/workflow.yamlのnodesに定義された各ノードを1ステップとして表示する。

**ステップ表示要素（1ステップ = 1カード）:**
```
┌──────────────────────────────┐
│ [状態アイコン] ステップ名    │
│ Agent: data_quality          │
│                    [詳細 ▼] │
└──────────────────────────────┘
```

**ステップ間の接続線:**
- 依存関係に基づいて矢印で接続
- 現在実行中のステップへの矢印はアニメーション（点滅）
- decision_branch_takenが存在する分岐は実際に選択されたパスを太線で表示

**ステップ詳細モーダル（クリックで展開）:**
```
タイトル: [ステップ名] の詳細

タブ構成:
  [入力] [出力] [ログ] [判断根拠]

入力タブ:
  JSON Viewer（コード表示、コピーボタン付き）
  schema_ref: cie://schemas/dataset.schema.json

出力タブ:
  実行完了後のみ表示
  JSON Viewer

判断根拠タブ:
  decision_branch_taken が存在する場合に表示
  例: normality_passed=true → welch_t_test を選択
```

### 実行中のタイムアウト表示

```
Rスクリプト実行中: ████████░░ 60%  (180秒 / 300秒上限)
                   ← 残り120秒以下でオレンジ、残り30秒で赤
```

```

14:32:15  data-quality → quality_gate         passed=true
14:32:31  statistics   → method_selected      welch_t_test
```

### Critical Issueカードの仕様

```
┌────────────────────────────────────────────────────────┐
│ ❌  [変数名] — [問題の種別]                            │  ← 赤背景
│                                                        │
│  [平易な日本語での説明（1〜2文）]                       │
│                                                        │
│  対処方法:                                             │
│  ① [具体的な操作A]                                    │
│  ② [具体的な操作B]                                    │
│                                                        │
│  [解消方法を確認]          [理解した上で進む]          │
│                              ↑ クリックで「承認済み」に │
└────────────────────────────────────────────────────────┘
```

**「理解した上で進む」クリック時の確認ダイアログ:**
```
この問題を解消せずに進みますか？

[変数名]の欠損率（34.2%）は推奨閾値（20%）を超えています。
この状態で解析を進めると、結果の信頼性が低下する可能性があります。

□ リスクを理解した上で進むことを確認します

[キャンセル]  [承認して進む]
```

### MissingValueChart仕様

- 横棒グラフ: X軸 = 欠損率(%)、Y軸 = 変数名（var_n表示）
- 20%閾値ラインを赤の縦破線で表示
- 5%閾値ラインをオレンジの縦破線で表示
- Critical（≥20%）: 赤バー
- Warning（5〜20%）: オレンジバー
- 正常（<5%）: 緑バー

### 右ペインの列名マッピングパネル

- デフォルト: オリジナル列名を`---`でマスク表示 + PIIパネルスタイル（UP-004）
- 「列名を表示する」ボタン:
  - Security Agentへr_code.restore_variablesリクエストを送信
  - 承認後: var_n → オリジナル列名に展開表示

- reviewerスコア < 90: 非活性 + 「品質スコアが90未満のためエクスポートできません」
- reviewerスコア ≥ 90: 活性化 → 承認パネル（UP-002）を表示
