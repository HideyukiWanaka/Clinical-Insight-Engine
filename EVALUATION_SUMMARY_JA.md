## フェーズ1.5: テストデータ作成 ✅ 完了

### データセット: `test_data/health_screening_japan.csv`

**仕様**:
- **レコード**: 190（150人のユニークな被験者 + 2年間の追跡40件）
- **列**: 25の医学/人口統計変数
- **言語**: 日本語（市場テストに現実的）
- **データ品質**: 現実的な欠損値（2-5%）、外れ値を含む

**主要変数**:

人口統計:
- 患者ID (PID00001-PID00150) — Layer 1 PII（正規表現パターン）
- 患者氏名 — Layer 2 PII（NLPセマンティック検出）
- 施設コード (FAC001-FAC003) — 施設識別子
- 検査年 (2024, 2025) — ペアード設計テスト用
- 年齢（30-75歳） — 年齢分布
- 性別（男性/女性） — 性別

生体指標:
- 身長_cm、体重_kg、BMI（人体測定）
- 収縮期血圧_mmHg、拡張期血圧_mmHg（血圧）

検査値:
- 空腹時血糖_mg/dl（空腹時血糖）
- HbA1c_%（グリケート化ヘモグロビン、約5%欠損）
- 総コレステロール_mg/dl（総コレステロール）
- HDLコレステロール_mg/dl、LDLコレステロール_mg/dl（脂質）
- 中性脂肪_mg/dl（トリグリセリド）
- AST_U/L、ALT_U/L、γ-GTP_U/L（肝機能、γ-GTP約5%欠損）
- クレアチニン_mg/dl（腎機能）
- eGFR_ml/min（推定糸球体ろ過量、約5%欠損）
- 尿酸_mg/dl（尿酸）
- 尿蛋白、尿糖（尿検査）

**データ品質特性**:
- 現実的な年齢/性別分布
- 生理学的に妥当な相関（BMI → 血糖、年齢 → 血圧）
- 臨床的外れ値（インデックス5でHbA1c 15.2%、インデックス10でクレアチニン2.8）
- 欠損値は実世界パターンを反映
- ペアード設計: 2024年と2025年の測定を持つ40人の被験者

**有効なテストシナリオ**:
1. ✅ 2群t検定（SBPの性別比較）
2. ✅ 相関分析（BMI vs HbA1c）
3. ✅ ペアード設計（HbA1c変化 2024→2025）
4. ✅ 重回帰（HbA1c ~ BMI + 年齢 + 空腹時血糖）
5. ✅ 品質ゲートブロック（外れ値にフラグ）
6. ✅ PII検出（Layer 2 NLP用患者名）

## フェーズ2: ランタイムテスト（保留中）

### フェーズ2の前提条件

6つの包括的テストシナリオを実行するには:

```bash
# 1. LLM APIキーをセット（必須）
export GOOGLE_GEMINI_API_KEY="AIza..."

# 2. オプション: 特定のプロバイダを設定

# 3. アプリケーションを起動
streamlit run cie/ui/app.py
```

### 実行するテストシナリオ

アプリが起動したら、これら6つのシナリオを順番に実行します:

| シナリオ | 臨床的質問 | 期待される手法 | 検証項目 |
|---------|----------|------------|---------|
| 1. 基本的なグループ比較 | 性別によるSBP差？ | t検定 | PL-001（意図）、記述統計、効果量 |
| 2. 相関分析 | BMI vs HbA1c相関？ | ピアソン/スピアマン | 欠損データ処理、可視化 |
| 3. **ペアード設計** | HbA1c変化(2024→2025)？ | ペアードt検定 | **PL-004検出**、**PL-005 subject_id** |
| 4. 多変量分析 | HbA1cを予測する要因？ | 重回帰 | VIFチェック、モデル診断 |
| 5. **品質ゲート** | フラグされたデータで分析？ | ユーザー決定 | **重大問題でブロック** |
| 6. **PII検出** | PII検出有効で分析？ | 削除/フラグ | **Layer 1正規表現 + Layer 2 NLP** |

### コアな機能

**1. 統計分析**
- 自然言語で臨床研究上の質問を認識

**2. データ品質管理**
- アップロードされたデータセットを検証
- 欠損値と外れ値を検出
- 臨床的に妥当でない値にフラグを立てる
- データ品質が重大な場合分析をブロック

**3. セキュリティとプライバシー**
- 分析前にPII（患者名、ID）を検出
- 機密データを削除またはニックネーム化
- すべての操作の監査証跡を保持
- Capability ベースのアクセス制御を実行

**4. ワークフロー自動化**
- マルチエージェント分析を編成
- 複雑な依存関係を処理（例：正規性 → 手法選択）
- ペアード設計と反復測定をサポート
- 出版準備が整ったレポートを生成

**5. 知識管理**
- 医学文献を摂取（PDF、テキスト）
